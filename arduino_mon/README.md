# UPS Reader (MIPS/Arduino)

Программа для опроса ИБП через COM-порт. Разработана специально для работы на встраиваемых устройствах (роутерах) под архитектуру MIPS (MIPSEL) в связке с Arduino на стороне ИБП.

## 🛠 Требования для сборки
Для компиляции необходим установленный кросс-компилятор под MIPS (обычно идет в составе OpenWrt SDK):
* `mipsel-unknown-linux-gnu-gcc`
* `mipsel-unknown-linux-gnu-strip`

## 📦 Сборка проекта

Программа собирается строго **статически** (`-static`). Динамическая сборка запрещена, так как на роутере используется библиотека `Musl libc`, а компилятор собран под `Glibc/uClibc` (динамическая сборка ломает структуру `termios` и вызывает ошибку `tcsetattr: Invalid argument`).

Чтобы скомпилировать и очистить бинарник:
```bash
make clean && make
```

После сборки утилита `strip` автоматически вырежет отладочные символы, уменьшив размер файла с ~4.3 МБ до **~540 КБ**.

## 🚀 Запуск и использование

Перенесите скомпилированный файл `ups_reader` на роутер (например, в `/tmp` или `/usr/bin/`) и запустите:
```bash
./ups_reader
```

### Формат вывода:
* При успешном опросе: `ONLINE BATTV=13.4 LINEV=230` (строка, которую прислала Arduino)
* При ошибке или таймауте: `OFFLINE BATTV=0.0 LINEV=0`

## ⚠️ Важные технические нюансы (почему это работает)

Если вы будете править код в будущем, помните три важные вещи:

1. **Защита от сброса Arduino (DTR)**: При открытии порта принудительно выставляется флаг `TIOCM_DTR` (`flags |= TIOCM_DTR`). Это удерживает линию DTR в высоком состоянии и не дает Arduino уходить в перезагрузку (Reset) при каждом запуске утилиты.
2. **Таймауты на стороне Ядра**: Вместо нестабильного на MIPS вызова `select()`, таймаут ожидания ответа (2 секунды) настроен прямо в драйвере через `tty.c_cc[VTIME] = 20`. Если Arduino молчит, `read()` корректно вернет `0` через 2 секунды.
3. **Очистка мусора**: Перед отправкой команды `Q\n` вызывается `tcflush(fd, TCIOFLUSH)`. Это стирает из буфера старые недочитанные байты от прошлых запусков.
